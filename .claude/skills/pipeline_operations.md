# Skill: Pipeline Operations

## Objective
Move leads through pipeline stages, log outreach activities (emails, calls, notes), and track stage changes.

## Tools Used
- `tools/activity_logger.py` — High-level activity logging functions
- `tools/db_manager.py` — SQLite CRUD for stages and activities

## Pipeline Stages (in order)
1. `new` — Initial/uncontacted
2. `researched` — Research completed
3. `email_sent` — Outreach email sent
4. `follow_up_1` — First follow-up
5. `follow_up_2` — Second follow-up
6. `responded` — Lead responded
7. `meeting` — Meeting scheduled
8. `proposal` — Proposal sent
9. `closed_won` — Deal closed (won)
10. `closed_lost` — Deal closed (lost)

## Available Functions

### From `activity_logger.py`
| Function | Purpose |
|----------|---------|
| `log_email_sent(db_path, lead_type, lead_key, description)` | Log email sent |
| `log_call_made(db_path, lead_type, lead_key, description)` | Log phone call |
| `log_note(db_path, lead_type, lead_key, description)` | Log a note |
| `change_stage(db_path, lead_type, lead_key, new_stage, description)` | Change stage + auto-log activity |

### From `db_manager.py`
| Function | Purpose |
|----------|---------|
| `get_lead_stage(db_path, lead_type, lead_key)` | Get current stage (defaults to `"new"`) |
| `set_lead_stage(db_path, lead_type, lead_key, stage)` | Set/update stage directly |
| `log_activity(db_path, lead_type, lead_key, activity_type, description)` | Low-level activity insert |
| `get_activities(db_path, lead_type, lead_key, activity_type, limit)` | Query activities with filters |

## Activity Types
- `email` — Email sent
- `call` — Phone call made
- `note` — Note/memo logged
- `stage_change` — Pipeline stage transition

## Notes
- Always use `change_stage()` from `activity_logger.py` rather than `set_lead_stage()` directly — it auto-logs the stage change as an activity
- `lead_type` is either `"company"` or `"contact"`
- `lead_key` is the 12-char hash ID generated by `excel_reader._generate_id()`
